# GBA 한글화 프로젝트 - Gemini(종합 정보) 검색 결과

## 개요
다양한 관점에서 수집한 GBA 게임 번역 및 로컬라이제이션의 종합 정보입니다.

## 1. Game Boy Advance 게임 개황

### GBA 플랫폼 특성
- **출시**: 2001년
- **프로세서**: ARM7TDMI CPU (16.78 MHz)
- **메모리**: ROM(32MB), RAM(32KB), VRAM(96KB)
- **해상도**: 240 × 160 픽셀

### 텍스트 저장 공간의 제약
- 제한된 ROM 크기로 인한 공간 제약
- 폰트 데이터 크기 최소화 필요
- 텍스트 압축 기법 필요할 수 있음

## 2. 게임 번역의 기본 개념

### 로컬라이제이션(Localization) vs 번역(Translation)
- **번역**: 단순히 언어를 바꾸는 것
- **로컬라이제이션**: 문화, 수치, 날짜, 통화 등 지역 맞춤화

### GBA 게임 번역 시 고려사항
1. **길이 제약**: 한글은 영어보다 짧은 경향
2. **폰트 제약**: 한글 글자 수가 많아 폰트 데이터 커짐
3. **메모리 제약**: ROM 크기 증가로 인한 다양한 연쇄 이슈
4. **게임 진행성**: UI, 메뉴 배치, 문자 열 제한 확인 필요

## 3. 번역 작업 프로세스

### Phase 4: 기술적 삽입

### Phase 5: QA 및 테스트

## 4. 게임 번역 도구 및 프로세스

### CAT (Computer-Assisted Translation)
- 전문 번역가를 위한 보조 소프트웨어
- **MemoQ**: 게임 번역에서 가장 일반적
- 용어 관리, 번역 메모리, 품질 보증 기능

### OCR 기반 도구
- **MORT**: 
  - PC 화면의 텍스트를 이미지로 캡처
  - OCR로 텍스트 추출
  - DB나 웹 번역기로 자동 번역
  - 게임 플레이 중 실시간 번역 가능

### 자동 번역 도구
- **XUnity.AutoTranslator**: 게임 자동 번역 도구
- 런타임 텍스트 가로채기(hooking)
- 온라인 번역 API 활용

## 5. 한글 인코딩 상세 분석

### EUC-KR (Extended Unix Code for Korean)
```
특징:
- KS X 1001, KS X 1003 표준 준수
- 2바이트(16bit) 고정 길이 인코딩
- ASCII와 호환성 유지
- 8,836자 표현 가능 (모든 한글 자모 미포함)
- 범위: 0x81 ~ 0xFE
```

### CP949 (Windows-949, Extended EUC-KR)
- EUC-KR 확장판
- 더 많은 한글 문자 지원
- Windows 환경에서 일반적

### UTF-8
```
특징:
- 가변 길이 인코딩 (1-4바이트)
- 모든 유니코드 문자 지원
- 한글: 3바이트 (예: 한글의 "한"은 0xED 0x95 0x9C)
- 최신 프로젝트에 권장
- 호환성 좋음
```

### GBA에서의 인코딩 선택
- **제약 환경**: EUC-KR (공간 절약)
- **유연성**: UTF-8 (더 많은 자 지원)
- 각 게임마다 상황에 맞게 결정
