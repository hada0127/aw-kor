### 6-4. 버그 추적 템플릿

```markdown
## 버그 보고서

### 기본 정보
- **버그 ID**: BUG-001
- **심각도**: Critical / Major / Minor / Enhancement
- **상태**: 발견 / 진행중 / 해결 / 확인
- **우선순위**: P0 / P1 / P2 / P3

### 증상
[버그의 구체적인 증상 기술]

### 재현 방법
1. [첫 번째 단계]
2. [두 번째 단계]
3. [세 번째 단계]

### 기대 결과
[정상적인 동작]

### 실제 결과
[발생한 문제]

### 스크린샷
[증거 자료]

### 영향 범위
- 게임 진행: 가능 / 불가능
- 영향 텍스트: [해당 텍스트/메뉴]
- 영향 기능: [해당 기능]

### 심각도 정의
- **Critical**: 게임 진행 불가, 데이터 손실, 심각한 충돌
- **Major**: 주요 기능 오류, 심각한 표시 오류
- **Minor**: 텍스트 오타, 작은 표시 오류
- **Enhancement**: 개선 제안
```

### 6-5. 버그 분류 및 우선순위

```
우선순위 설정:

P0 (긴급):
  - 게임 진행 불가
  - 데이터 손실 위험
  - 심각한 충돌
  → 즉시 해결 필요

P1 (높음):
  - 주요 기능 오류
  - 심각한 표시 오류
  - 게임플레이 영향
  → 이 주 내 해결

P2 (중간):
  - 사소한 표시 오류
  - 오타
  - 개선 제안
  → 다음 주 내 해결

P3 (낮음):
  - 미래 버전 개선
  - 엔하ンスment
  → 가능하면 해결

분류:
  Critical: P0, P1
  Major: P1, P2
  Minor: P2, P3
  Enhancement: P3
```

## 리스크 및 대응 방안

| 리스크 | 확률 | 영향 | 대응 |
|-------|------|------|------|
