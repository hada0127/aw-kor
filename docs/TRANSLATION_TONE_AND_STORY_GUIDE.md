## 1. 게임 기본 정보

### 게임 특성
- **제목**: Game Wars 1+2
- **장르**: 전술 전쟁 게임 (Tactical Strategy Game)
- **배경**: 현대 군사 시나리오
- **플레이 방식**: 턴제 전술 배틀

### 게임 구성 요소
- 다양한 해군/육군/공군 유닛 시스템
- 턴 기반 전투 메커닉
- 유닛 이동, 공격, 방어, 특수능력 사용
- 자원 관리 및 전략 수립

## 2. 톤앤매너 (Tone and Manner)

### 2-1. 전체 문체 특징

#### 핵심 원칙
```
군사 게임의 딱딱한 문체
- 직설적이고 명확함
- 전문 용어 사용 (군사 용어)
- 형식적이지만 친숙함
- 게임 메뉴답게 간결함
```

#### 문어 특징
- **높임/존댓말 혼합**: 게임 UI이므로 존댓말 형식 사용
- **시제**: 현재형 기본
- **종료표현**: 체언+은/는, 동사+다/다. 형식
- **감정표현**: 최소화 (게임의 특성상 객관적)

### 2-2. 카테고리별 번역 규칙

#### A. 군사 유닛명 번역

**규칙**
```
- 한자어(중국식 한글) 우선 사용
- 고유명사이므로 일관성 필수
- 원문의 기술적 정보 유지
- 게임 내 모든 위치에서 동일하게 번역
```

**현재까지 번역된 유닛들** (완전 일관성 확인)

| 일본어 | 한글 | 설명 | 빈도 |
|--------|------|------|------|
| 潜水艦 | 잠수함 | 잠수하는 전함 | 매우 높음 |
| 輸送船 | 수송함 | 군사용 수송선 | 높음 |
| 護衛艦 | 호위함 | 호위하는 전함 | 높음 |
| 輸送ヘリ | 수송헬기 | 헬기로 수송 | 높음 |
| 戦闘ヘリ | 전투헬기 | 전투용 헬기 | 높음 |
| 爆撃機 | 폭격기 | 폭격 전문 항공기 | 높음 |
| 対空戦車 | 대공전차 | 대공 방어용 전차 | 높음 |
| ロケット砲 | 로켓포 | 로켓 발사 포 | 높음 |
| 自走砲 | 자주포 | 자력주행 포 | 높음 |
| 軽戦車 | 경전차 | 가벼운 전차 | 높음 |
| 重戦車 | 중전차 | 무거운 전차 | 높음 |
| バズーカ兵 | 바주카병 | 바주카(로켓 런처) 사용 병사 | 중간 |

**주요 특징**
- 모든 유닛명이 **완벽하게 일관성 있게** 번역됨
- 한자 기반의 정확한 한글표현
- 게임 내 모든 참조에서 동일한 표기

#### B. 액션/능력 표현

**규칙**
```
- 게임 액션은 명사형 + 동사형 병행
- 능력 판정 표현: ~할 수 있다. / ~할 수 없다.
- 높은 게임성 유지 (따라 하기 쉬운 표현)
- 전투 중 빠른 인식이 가능한 문구
```

**현재 번역 표현**

| 일본어 | 한글 | 사용 상황 |
|--------|------|---------|
| できる。 | 할 수 있다. | 행동 가능 상태 |
| できない。 | 할 수 없다. | 행동 불가능 상태 |

**특징**
- 마침표 포함 (문장의 종료 명확)
- 선택지나 상태 표현에 사용
- 게임플레이 흐름에 맞는 간결한 표현

#### C. UI/메뉴 텍스트

**규칙**
```
- 게임 메뉴답게 짧고 명확
- 1-3 단어로 표현 (메뉴 폭 제한)
- 명령형 또는 명사형 사용
- 사용자 이해 최우선
```

**예상 번역 (계획)**
- スタート = 시작
- コンティニュー = 계속
- 難易度 = 난이도
- 攻撃 = 공격
- 移動 = 이동
- 防御 = 방어

### 2-3. 길이 최적화 (Localization)

**규칙**
```
원문 길이와 한글 길이의 균형
- 원문과 유사: 최적
- 원문보다 짧음: 허용
- 원문보다 길음: 최소화 (화면 오버플로우 주의)
```

**현재 통계**
```
평균 번역 길이: 약 8-10 bytes (한글 3-5자)
최소: 6 bytes (한글 2자)
최대: 10 bytes (한글 5자)

예) 
- 戦闘ヘリ (8 bytes) → 전투헬기 (8 bytes) ✓
- バズーカ兵 (10 bytes) → 바주카병 (10 bytes) ✓
```

### 2-4. 문체 금지사항

```
✗ 존댓글 스타일: "합니다", "됩니다" (게임 UI 부적절)
✗ 너무 딱딱한 표현: "~이다", "~를 한다" (게임성 떨어짐)
✗ 비속어/비공식 표현 (게임 품질 하락)
✗ 일본식 한글 직역 (자연스럽지 않음)
✗ 영문 혼용 (한글화 목적 위반)
```

## 3. 게임 스토리 및 설정

### 3-1. 게임의 전체 배경

**시대**: 현대 (또는 가까운 미래)  
**무대**: 국제 전쟁 시나리오  
**주요 세력**: 여러 국가/진영  

### 3-2. 핵심 스토리 요소

#### 등장하는 유닛 분석
현재 번역된 유닛들로부터 추론:

**해군 (Naval Forces)**
- 잠수함 (Submarine)
- 수송함 (Transport Ship)
- 호위함 (Escort Ship)

→ **스토리 함의**: 해상 수송로 확보, 잠수전 전개 등 해양 전략 중요

**항공 (Air Forces)**
- 수송헬기
- 전투헬기
- 폭격기

→ **스토리 함의**: 공중 우위 확보, 전략 폭격, 헬기 기동전 등

**지상군 (Ground Forces)**
- 경전차 / 중전차
- 대공전차
- 자주포
- 로켓포
- 바주카병

→ **스토리 함의**: 지상 영토 확보, 대공 방어, 포병 전 등

### 3-3. 게임플레이 시나리오

**추론된 게임 흐름**

```
1. 전투 준비 단계
   - 유닛 배치 선택
   - 난이도 설정
   - 전술 계획

2. 전투 진행 단계
   - 턴 기반 이동
   - 유닛별 행동 (이동/공격/방어)
   - 능력 판정 (할 수 있다/없다)

3. 승리/패배 판정
   - 목표 달성
   - 전사 유닛 처리
   - 다음 스테이지 진행
```

### 3-4. 캐릭터/세력

**현재까지 명시된 캐릭터**: 거의 없음 (군사 게임의 특성상)  
**대신 강조되는 요소**:
- 유닛의 성능과 특성
- 전술적 판단
- 자원 관리

→ **스토리 특징**: 개인 캐릭터보다 대국적 전쟁 시나리오 중심

### 3-5. 게임의 핵심 메시지

**예상되는 테마**
```
1. 전술과 전략의 중요성
2. 다양한 유닛 조합의 활용
3. 제한된 자원으로 최대 효과 달성
4. 위험 대비 (방어의 중요성)
```

## 4. 번역 가이드라인

### 4-1. 군사 용어 번역 원칙

**최우선**: 정확성과 일관성
```
- 같은 일본어 = 항상 같은 한글
- 군사 전문 용어 → 인정된 한글 표기 사용
- 신조어 → 게임 커뮤니티에서 통용되는 표현
```

**추천 참고 자료**
- 국방부 군사 용어집
- 게임 커뮤니티 통용 표현
- 기존 GBA 한글화 프로젝트

### 4-2. 문체 일관성 체크리스트

번역 전 확인사항:
```
□ 게임 UI에 어울리는가?
□ 같은 표현은 항상 동일하게 사용되었는가?
□ 군사 용어가 정확하게 번역되었는가?
□ 길이가 화면에 맞는가? (오버플로우 없나?)
□ 비속어나 부적절한 표현은 없나?
□ 일본식 직역이 아닌가?
```

### 4-3. 우선 순위

**Priority 1: UI/메뉴 텍스트** (게임 진행에 필수)
- 시작, 계속, 설정 등
- 유닛명
- 행동 (이동, 공격, 방어)

**Priority 2: 스토리/대사** (게임 이해에 필수)
- 시나리오 설명
- 유닛 상태 메시지
- 전투 진행 안내

**Priority 3: 기타 텍스트** (선택사항)
- 아이템 설명
- 상세 정보
- 팁/가이드

## 8. 번역 시 주의사항

### 주의 1: 길이 제약

많은 게임 메뉴는 화면 폭이 정해져 있음
```
주의: 원문보다 훨씬 길면 화면 오버플로우 발생
해결: 짧은 동의어 사용 또는 약자 활용

예) 轻戦車 (가벼운 전차) → 경전차 (축약)
```

### 주의 2: 일관성 유지

같은 단어는 항상 같게 번역
```
✓ 潜水艦 = 항상 "잠수함" (다른 표현 금지)
✓ 攻撃 = 항상 "공격" 
✓ 移動 = 항상 "이동"
```

### 주의 3: 게임 문화 존중

GBA 게임의 특성 이해
```
- 한정된 화면 크기
- 빠른 진행 속도
- 명확한 정보 전달 필수
- 복잡한 설명보다는 간결함
```
