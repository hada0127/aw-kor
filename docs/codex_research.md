## 2. 테이블 파일(Table File, .tbl) 시스템

### .tbl 파일이란?
- 텍스트 테이블 파일로 ROM의 텍스트 형식을 읽을 수 있게 변환
- Hex 편집기가 텍스트를 제대로 표시하고 편집할 수 있게 함

### .tbl 파일 형식
```
# 주석은 # 기호로 시작
# 각 라인은 HEX=string 형식
00=A
01=B
02=C
```

### .tbl 파일 생성 도구
- **RSEARCH**: 상대적 검색을 통한 테이블 파일 생성
- **Monkey-Moore**: 테이블 파일 빌드 도구
- **WindHex**: TBL 지원 Hex 편집기

### .tbl 파일 활용 방법
1. 게임의 텍스트 인코딩 분석
2. 상대적 검색 도구로 인코딩 매핑 추출
3. .tbl 파일 생성
4. Hex 편집기에서 로드하여 텍스트 읽기/편집

## 3. 문자열 추출 및 포인터 수정

### 문자열 추출 방법
1. **Hex 스캔**: ROM을 16진수로 읽고 문자열 위치 파악
2. **테이블 파일 활용**: .tbl로 문자열 자동 디코딩
3. **에뮬레이터 디버깅**: 메모리 접근으로 동적 분석

### 포인터 시스템
- GBA 게임의 대사는 메모리 포인터로 참조됨
- 한글 문자열 길이 변화에 따라 포인터 업데이트 필요
- 32비트 주소 포인터 (4바이트, 리틀엔디안)

### 포인터 수정 도구
- Hex 편집기로 직접 수정
- 자동화 스크립트로 일괄 처리
- 컴파일 시 자동으로 포인터 재계산

## 5. ROM 해킹 워크플로우

### 기본 과정
1. **분석 단계**
   - ROM 구조 파악
   - 문자열 위치 찾기
   - 인코딩 방식 결정

2. **추출 단계**
   - .tbl 파일 생성
   - 모든 텍스트 추출
   - 메타데이터 수집

3. **편집 단계**
   - 번역 진행
   - 길이 조정
   - 포인터 업데이트

4. **삽입 단계**
   - 번역된 텍스트 삽입
   - 폰트 데이터 추가
   - 체크섬 재계산

5. **테스트 단계**
   - 에뮬레이터에서 검증
   - 그래픽 오버플로우 확인
   - 게임 진행성 테스트

## 6. 인코딩 및 폰트 관리

### 한글 인코딩 옵션
- **EUC-KR**: 2바이트 완성형 인코딩, 8,836자 지원
- **CP949(확장 EUC-KR)**: 보다 많은 한글 자 지원
- **UTF-8**: 가변길이(1-4바이트), 모든 유니코드 지원

### 인코딩 선택 기준
- 기존 프로젝트: 기존 인코딩 유지 (EUC-KR 등)
- 신규 프로젝트: UTF-8 권장 (확장성)
- GBA 게임: 공간 제약으로 EUC-KR 또는 커스텀 인코딩 많음

### 폰트 데이터 관리
- GBA 내 폰트 데이터 위치 파악
- 한글 글리프(glyph) 생성 또는 추출
- 폰트 메모리 크기 제약 관리
