## 2. 필수 번역 규칙

### 규칙 1: 군사 유닛명 - 절대 일관성

**현재까지 번역된 유닛 사전** (완벽한 일관성):

```
Naval Forces (해군):
  潜水艦 → 잠수함 (submarine)
  輸送船 → 수송함 (transport ship)
  護衛艦 → 호위함 (escort ship)

Air Forces (공군):
  輸送ヘリ → 수송헬기 (transport helicopter)
  戦闘ヘリ → 전투헬기 (combat helicopter)
  爆撃機 → 폭격기 (bomber aircraft)

Ground Forces (육군):
  軽戦車 → 경전차 (light tank)
  重戦車 → 중전차 (heavy tank)
  対空戦車 → 대공전차 (anti-air tank)
  ロケット砲 → 로켓포 (rocket artillery)
  自走砲 → 자주포 (self-propelled gun)
  バズーカ兵 → 바주카병 (bazooka soldier)
```

**중요**: 위 사전에 없는 새로운 유닛이 나올 경우:
1. 같은 패턴 적용 (한자어 기반)
2. 게임 커뮤니티 통용 표현 참고
3. 일관성 유지

### 규칙 2: 액션/능력 표현 - 표준형식

```
가능 상태:
  できる。 → 할 수 있다.

불가능 상태:
  できない。 → 할 수 없다.
```

**중요**: 마침표 포함, 높은 게임성 유지

### 규칙 3: 길이 최적화

```
목표: 원문과 비슷하거나 더 짧게

측정 기준: Byte 수 (UTF-8 기준)
  - 한글 1자 = 약 3 bytes
  - 한글 2자 = 약 6 bytes
  - 한글 3자 = 약 9 bytes

예시:
  戦闘ヘリ (8 bytes) → 전투헬기 (8 bytes) ✓
  バズーカ兵 (10 bytes) → 바주카병 (10 bytes) ✓
  潜水艦 (6 bytes) → 잠수함 (6 bytes) ✓
```

**주의**: 화면 오버플로우 방지 (GBA 화면 제약)

### 규칙 4: 톤앤매너 - 게임 스타일

```
특징:
  - 군사 게임의 딱딱한 문체
  - 직설적이고 명확함
  - 형식적이지만 친숙함
  - 존댓말 기본

금지사항:
  ✗ "합니다", "됩니다" (존댓글 - 게임 UI 부적절)
  ✗ "~이다" (너무 딱딱함)
  ✗ 비속어 (게임 품질 하락)
  ✗ 일본식 직역 (자연스럽지 않음)
  ✗ 영문 혼용 (한글화 위반)

허용사항:
  ✓ "한다", "한다.", "한다!" (명령/상태)
  ✓ "~할 수 있다/없다" (능력 표현)
  ✓ 한자어 (군사 용어)
  ✓ 간결한 표현 (메뉴 최적화)
```

## 5. 일반적인 오류와 해결

### 오류 1: 유닛명 불일치

```
✗ 잘못된 예:
  전차 (다른 곳에선 "전차" 사용)
  대공전 (전체적으로는 "대공전차")

✓ 올바른 예:
  항상 "대공전차" 사용
  사전 기준 유지
```

**해결**: 제공된 유닛 사전 참고

### 오류 2: 길이 초과

```
✗ 잘못된 예:
  爆撃機 (8 bytes) → 폭격전투기 (12 bytes) ✗

✓ 올바른 예:
  爆撃機 (8 bytes) → 폭격기 (6 bytes) ✓
```

**해결**: 동의어 축약, 한글 특성 활용

### 오류 3: 톤 부적절

```
✗ 잘못된 예:
  "공격 행동을 수행할 수 있습니다" (존댓말)
  "쩌렁하게 공격하기" (비속어)

✓ 올바른 예:
  "공격할 수 있다." (게임 톤)
```

**해결**: TRANSLATION_TONE_AND_STORY_GUIDE.md 참고

## 6. 데이터 형식 상세

### CSV 입력 형식

```
address,japanese,korean,length

예시:
0x009294D0,潜水艦,잠수함,6
0x00A030B8,"でも,","하지만,",6
0x00A036D4,・・・,···,6
```

**필드 설명**:
- `address`: ROM 내 텍스트 위치 (16진수)
- `japanese`: 원문 (쉼표 포함 시 따옴표)
- `korean`: 한글 번역 (쉼표 포함 시 따옴표)
- `length`: 한글 길이 (UTF-8 bytes)

### Length 계산 방법

```python
korean_text = "잠수함"
length = len(korean_text.encode('utf-8'))  # 6 bytes
```

## 8. 문제 해결 (Troubleshooting)

### 문제 1: "문자 렌더링 오류"

원인: EUC-KR 인코딩 오류 또는 잘못된 길이 정보

**해결**:
```python
# 정확한 길이 계산
korean = "잠수함"
correct_length = len(korean.encode('euc-kr'))  # 6
# length 필드에 정확한 값 입력
```

### 문제 2: "유닛명이 게임에서 다르게 표시됨"

원인: 사전과 다른 표현 사용

**해결**:
1. `.project-config.json`의 unit 사전 확인
2. `TRANSLATION_TONE_AND_STORY_GUIDE.md` 참고
3. 모든 유닛명 통일

### 문제 3: "텍스트가 화면 밖으로 나감"

원인: 길이가 너무 길어서 오버플로우

**해결**:
1. 더 짧은 동의어 찾기
2. 약자 사용 고려
3. 원문과 비슷한 길이 목표
