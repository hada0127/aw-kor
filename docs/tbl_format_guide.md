# Game Wars .tbl 파일 형식 가이드

## .tbl 파일이란?

`.tbl` (Table) 파일은 **바이트와 문자 간의 맵핑** 을 정의하는 텍스트 파일입니다.
ROM 해킹에서 텍스트를 추출하고 삽입할 때 사용됩니다.

## 기본 형식

```
<16진수>=<문자>
```

**예시:**
```
00=あ
01=い
02=う
20= 
FF=\end
```

## 상세 규격

### 1. 16진수 부분

- **범위**: 00 ~ FF (0 ~ 255)
- **표기**: 대소문자 상관없음 (00, 0x00 같은 형식 사용 불가)
- **선행 0**: 필수 (예: "0A" 아닌 "A")

**올바른 예:**
```
00=a
0A=b
A0=c
FF=end
```

**잘못된 예:**
```
0=a          # 선행 0 없음
0x0A=b       # 0x 접두사 사용
A=c          # 선행 0 없음
```

### 2. 문자 부분

#### 2.1 일반 문자

단일 바이트 맵핑:
```
30=0
31=1
41=A
61=a
```

#### 2.2 멀티바이트 문자

일본어의 히라가나, 가타카나는 보통 2~3바이트:

```
# 히라가나 (2바이트 Shift-JIS 예시)
8142=あ
8144=い
8146=う
```

#### 2.3 특수 문자 (이스케이프)

제어 문자는 이스케이프 시퀀스로 표현:

| 문자 | 표현 | 설명 |
|------|------|------|
| 개행 | `\n` | 줄바꿈 |
| 탭 | `\t` | 탭 문자 |
| 널 | `\0` | 문자열 끝 |
| 공백 | ` ` | 일반 공백 |
| 백슬래시 | `\\` | 백슬래시 문자 |

**예시:**
```
00=\0       # 널 종료 문자
0A=\n       # 개행
09=\t       # 탭
20= 	     # 공백
5C=\\       # 백슬래시
FF=\end     # 특수 표시
```

#### 2.4 마크업 및 특수 표시

일부 게임은 특수 문자나 마크업 태그 사용:

```
# 색상 변경
F0=\RED
F1=\GREEN
F2=\BLUE

# 일시 정지
FE=\WAIT

# 문자열 끝
FF=\end
```

### 3. 주석

`#` 기호로 시작하는 줄은 무시됩니다:

```
# 히라가나
00=あ
01=い

# 가타카나
02=ア
03=イ
```

## 검증 방법

### 1. HxD에서 로드

1. HxD 설정에서 .tbl 파일 로드
2. ROM을 다시 엽니다
3. 텍스트가 올바르게 표시되는지 확인

### 2. Python으로 검증

```python
def validate_tbl(tbl_path, rom_path):
    """
    .tbl 파일의 일관성을 검증합니다
    """
    mapping = load_tbl(tbl_path)
    
    # 중복 검사
    chars = list(mapping.values())
    if len(chars) != len(set(chars)):
        print("경고: 중복된 문자 맵핑")
    
    # 범위 검사
    for byte_val in mapping.keys():
        if not (0 <= byte_val <= 255):
            print(f"오류: 범위 벗어난 바이트: {byte_val}")
```

## 일반적인 문제

### Q: 같은 문자가 여러 바이트로 맵핑됩니다

**A**: 인코딩 문제일 수 있습니다. 
- 가장 많이 나타나는 바이트값을 선택
- 또는 각 컨텍스트마다 다른 테이블 생성

### Q: 특정 텍스트가 표시되지 않습니다

**A**: 
- 해당 문자의 바이트값을 확인
- ROM에서 같은 바이트값으로 검색
- 맵핑이 정확한지 재확인

### Q: 멀티바이트 문자가 제대로 인식되지 않습니다

**A**:
- 인코딩 방식 확인 (Shift-JIS, EUC-JP 등)
- 바이트 순서 확인 (big-endian vs little-endian)
- 게임 개발사 문서 참고
