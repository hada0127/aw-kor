# Game Wars GBA 번역 프로세스 가이드

이 가이드는 Game Wars GBA 한글화를 위한 번역 프로세스를 설명합니다.

## 개요

번역은 다음 단계로 진행됩니다:

1. **텍스트 추출** (PHASE 3) → CSV 파일로 정렬
2. **번역팀 구성** (PHASE 4) → 역할 분담
3. **번역 진행** → 우선순위별로 진행
4. **검수 및 수정** → 품질 관리
5. **최종 검증** → ROM 삽입 전 확인

## 1. 텍스트 추출 및 준비

### 1.1 CSV 파일 형식

`data/game_wars_1_text.csv`:

```csv
address,japanese,length,context,priority,translated_korean,translator,status,notes
0x200000,"あなたの名前",6,"캐릭터 이름 입력",1,"당신의 이름","홍길동","완료","메인 캐릭터"
0x200010,"おはようございます",12,"인사 대사",2,"좋은 아침입니다.","김영희","검수 중","자연스러운 톤"
0x200022,"ゲーム開始",6,"메인 메뉴",1,"게임 시작","이순신","완료",""
```

### 1.2 필드 설명

| 필드 | 설명 |
|------|------|
| `address` | ROM에서의 주소 (16진수) |
| `japanese` | 원문 (일본어) |
| `length` | 바이트 길이 |
| `context` | 텍스트가 나타나는 게임 내 상황 |
| `priority` | 번역 우선순위 (1=최우선, 3=낮음) |
| `translated_korean` | 한글 번역 |
| `translator` | 번역가 이름 |
| `status` | 상태 (미번역/번역중/완료/검수중/검수완료) |
| `notes` | 특수사항, 톤앤매너 등 |

### 1.3 Google Sheets로 공유

CSV를 Google Sheets로 변환하여 팀과 공유:

1. Google Sheets 생성
2. CSV 파일 import
3. 공유 링크 생성 및 팀 배포

### 1.4 번역 우선순위 설정

```
Priority 1 (필수, 1-2주):
  - 메인 메뉴 항목
  - UI 텍스트 (HP, MP 등)
  - 게임 플레이에 필수적인 메시지

Priority 2 (중요, 2-4주):
  - 메인 스토리 캐릭터 대사
  - 전투 메시지
  - 튜토리얼 텍스트

Priority 3 (추가, 1-3개월):
  - 아이템 설명
  - 도움말 텍스트
  - 배경 스토리
```

## 2. 번역팀 구성

### 2.1 역할 정의

#### 리더 (Team Lead)
- 프로젝트 감독
- 진행률 관리
- 분쟁 해결
- 최종 검증

#### 번역가 (2-3명)
- 실제 번역 작업
- 톤앤매너 일관성 유지
- 게임 컨텍스트 이해

#### 검수자 (1-2명)
- 번역 품질 검증
- 문법/철자 확인
- 게임 플레이 검증
- 폰트 오버플로우 확인

#### 기술 담당자 (1명)
- ROM 분석 및 포인터 매핑
- 텍스트 삽입 및 ROM 빌드
- 인코딩 관련 기술 지원

### 2.2 역할 분담 예시

```
Game Wars 1:

  - 메인 스토리: 이순신 (번역가)
  - 아이템/상태: 박문수 (번역가)

  - (Game Wars 1 팀과 병렬 진행)
```

## 3. 번역 진행 관리

### 3.1 용어집 (Glossary) 작성

번역 일관성을 위해 핵심 용어를 정의합니다:

```csv
영문 용어,일본어,한글,설명,우선순위
Unit,ユニット,유닛,게임 내 캐릭터/병사,1
HP,ヒットポイント,체력,생명력 수치,1
MP,マジックポイント,마나,마법 사용 능력치,1
Move,移動,이동거리,한 턴에 이동할 수 있는 칸 수,1
Attack Power,攻撃力,공격력,데미지 수치,1
Defense,防御力,방어력,받는 데미지 감소,1
```

### 3.2 주간 진행 체크

매주 진행률 확인:

```
Week 1:
  - Priority 1 (필수): 0/50 → 20/50 (40%)
  - 예상: 50개 완료

Week 2:
  - Priority 1: 20/50 → 50/50 (100%)
  - Priority 2: 0/200 → 50/200 (25%)
  - 누적: 50개

Week 3:
  - Priority 2: 50/200 → 100/200 (50%)
  - 누적: 100개
```

### 3.3 번역 작업 프로세스

```
CSV에서 미번역 항목 선택
        ↓
번역 작성 (Translator)
        ↓
자신의 상태를 "번역중"으로 표시
        ↓
완료 후 "완료"로 표시
        ↓
검수자가 "검수중"으로 변경
        ↓
검수 완료 후 "검수완료"로 표시
        ↓
다음 번역가가 진행
```

### 3.4 게임 내 문맥 확인

각 번역가는:

1. Game Wars를 에뮬레이터로 실행
2. 해당 텍스트가 나타나는 시점 찾기
3. 게임의 톤앤매너에 맞는 번역 작성
4. 스크린샷과 함께 기록

예:
```
주소: 0x200000
원문: "あなたの名前"
위치: 게임 시작 시 캐릭터 이름 입력 화면
톤앤매너: 정중한 존댓말
한글: "당신의 이름"
스크린샷: data/screenshots/name_input.png
```

## 4. 번역 검수 가이드

### 4.1 검수 체크리스트

번역된 각 항목을 검수할 때:

- [ ] 원문 의미를 정확히 전달하는가?
- [ ] 게임의 톤앤매너와 일치하는가?
- [ ] 한글이 자연스러운가?
- [ ] 오타나 문법 오류는 없는가?
- [ ] 길이가 화면에 맞는가? (ROM에 삽입 가능한가?)
- [ ] 용어가 용어집과 일치하는가?
- [ ] 숫자나 고유명사는 정확한가?

### 4.2 길이 검증

한글화 시 중요한 고려사항:

```
일본어: "ユニット" (6바이트)
한글: "유닛" (4바이트 EUC-KR)

할당된 ROM 공간이 6바이트면:
  → 한글 변환: "유닛" (4바이트)
  → 여유: 2바이트 (패딩 가능) ✓

할당된 ROM 공간이 4바이트면:
  → 한글 변환: "유닛" (4바이트)
  → 여유: 0바이트 ✓ (딱맞음)

할당된 ROM 공간이 2바이트면:
  → 한글 변환 불가능 ✗
  → 더 짧은 단어 필요
```

### 4.3 특수 문자 처리

게임에서 사용되는 특수 표기:

```
\n = 개행
\t = 탭/들여쓰기
\0 = 문자열 끝

예:
"ゲーム開始\n新規ゲーム\n続きから"
→
"게임 시작\n새로운 게임\n이어하기"
```

## 5. 번역 분쟁 해결

### 5.1 용어 분쟁

```
예: "ユニット"을 "유닛" vs "부대"로 번역

리더의 결정:
  - 게임 내 맥락 확인
  - 다른 게임 번역 관례 참고
  - 최종 결정: "유닛" (게임 통상 용어)
  - 모든 참여자에게 공지
  - 용어집 업데이트
```

### 5.2 톤앤매너 분쟁

```
예: "おはようございます"을 
  A안: "좋은 아침입니다" (정중)
  B안: "아침이야" (캐주얼)
  C안: "좋은 아침이에요" (친근)

리더의 결정 프로세스:
  1. 게임 장르/톤 확인 (전략 게임 → 정중)
  2. 캐릭터 성격 확인
  3. 다른 유사 게임 참고
  4. 최종 선택: A안 (정중) - 게임 톤 부합
```
